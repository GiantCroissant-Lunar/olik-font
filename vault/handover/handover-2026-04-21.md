## Project conventions (authoritative — see `.agent/skills/INDEX.md` § Project-level conventions)

- **Archon uses Codex SDK only** for AI nodes. No Claude Code SDK. Non-Codex assistants (Pi / Kimi / Copilot) run via `bash:` nodes.
- **Archon backing store is PostgreSQL**, not SQLite.
- **One CLI, one job.**
  - Pi = ZAI / GLM only (even though upstream Pi supports many providers)
  - Codex = OpenAI + Archon's SDK backend
  - Kimi = Moonshot
  - Copilot = optional GitHub-native shell helpers
  - Ollama = Gemma 4 (`:31b`, cloud-routed)
- **Gemma 4 is cloud-only for 31b.** 31b is too large to run locally; `ollama signin` routes to cloud. Smaller tags could run locally but are too weak. **Never** `ollama pull gemma4:31b`.
- **OAuth-only for Codex and Copilot.** Both accept env-var auth upstream; we forbid it here — keeps plaintext keys out of `.env` and rotation tied to the subscription.
- **No LiteLLM / no Mastra.** Pi already unifies ZAI; Archon handles workflows. Adding a router is pure overhead on current scope.
- **`ref-projects/` is read-only.** Gitignored. Never edit, stage, or commit anything under it.
- **Secrets in `infra/.env`.** Only Pi (ZAI), Ollama, and the optional Kimi SDK fallback need env entries.
