# Olik-font versioned roadmap

Framed as explicit v1/v2/v3/vN milestones so new failure modes are assigned
to a version rather than dismissed as "structural / out of scope". Captures
the handover-state critique that the project kept stopping at v1 without
sketching what comes after.

## Guiding rule (carried across every version)

`.agent/skills/05-engine/glyph-geometry/SKILL.md` — component placement is a
measured affine transform in 1024² virtual space. Never preset buckets,
never global slot weights, never categorical flags that patch preset
behaviour. This rule does not weaken between versions.

## v1 — Correct reconstruction for the decomposable core (DONE)

Scope:
- 4 seed chars 明 / 清 / 國 / 森 render MMH-faithful.
- Scale to 100+ Chinese chars whose cjk-decomp arity matches MMH's
  `matches` partition arity AND whose host + components all live in MMH
  graphics.

Delivered by Plan 11 (commits `021377e` → `e6b82f9`, tag
`plan-11-virtual-coordinates-only`):
- Preset vocabulary deleted from code, data, and emitted JSON.
- `compose/walk.py` measures transforms from MMH source strokes.
- `bulk/planner.py` drives placement from MMH `matches` partition.
- Op-whitelist gate removed (operator name is metadata, not placement).

At-scale state after v1: verified=127, needs_review=5, unsupported_op=0,
failed_extraction=64 across 196 filled / 4808. Sample of 10 scaled-up
chars: IoU=1.000 each. Kimi-CLI visual verdict on 6 random scaled-up
chars: `pass` (6/6 composed ↔ MMH matching).

## v2 — Raise the variant cap and clean up the remaining IoU edge cases

Scope (small; mostly data tuning):
- 5 chars in the `needs_review` bucket after v1: 匠 威 式 街 褻. These
  compose but hit `0.85 ≤ iou_mean < gate`, typically because a
  component's variant cap hit before the correct variant was minted.
  Raising `--max-variants-per-proto` from the default 2 to 10 already
  cleared 32 of 36 historical needs_review cases; the remaining 5 may
  need per-prototype cap tuning or a second-pass matcher.
- Explicit per-prototype cap config (instead of a single global `--cap`),
  so high-frequency components (木, 口, 日) can carry 10+ variants while
  rare components stay at 2.

Exit criteria: at least 130 / 196 filled verified at cap=10 on this
random 196 sample. Zero `unsupported_op`; `needs_review` ≤ 3.

## v3 — Recursive decomposition to handle cjk-deeper-than-MMH

Scope:
- 29 chars in the `A: cjk component count > MMH partition arity` bucket
  (丐 乙 兆 刁 剎 力 …). cjk-decomp gives 2–5 components; MMH's top-level
  `matches` gives only 1–2 partitions. These chars need recursive
  descent: use `nested_partition(matches, top_level_idx=k)` (already
  shipped in `bulk/mmh_partition.py`) to find sub-component stroke
  assignments.
- Extend `planner.plan_char` to detect arity mismatch and try nested
  levels of `matches` before giving up. 森 is the working blueprint
  (MMH has nested buckets `[1,0]`, `[1,1]` under component 1) — the
  same mechanism applies to other chars.
- Where MMH's nesting is still shallower than cjk-decomp, fall through
  to v4 data extension.

Exit criteria: the 29-char "A" bucket drops to ≤ 5.

## v4 — Extend beyond MMH using ref-projects data sources

Scope: lift the self-imposed "MMH is ground truth" cap. The seed spec
explicitly says MMH is a starting point, not a boundary. Three
extensions, each maps 1:1 to a failure class:

- **Extend host graphics coverage** (fixes class C: 7 chars — 縲 臏 蓀
  迆 鉑 鍰). Add a merged graphics loader that tries MMH first, then
  animCJK's `graphicsZhHant.txt` (and `dictionaryZhHant.txt`) at
  `ref-projects/animCJK/`. animCJK ships ~4808 traditional-Chinese
  entries in MMH-compatible stroke format.
- **Extend component graphics** (fixes class D: 28 chars — 乏 侖 前 叉
  商 天 …). Components that aren't standalone glyphs (like ⺊, ㇐, ㇒)
  need their strokes carved from a containing char. Mechanism: when
  `mmh[component_name]` is missing, look the component up in cjk-decomp
  as a leaf-of-leaf under a real host, then measure its strokes through
  the partition chain.
- **Extend partition source** (helps class A when MMH `matches` is
  incomplete). animCJK's `acjk` string is an encoded per-stroke
  component-index; write an `acjk_partition(acjk, decomp)` adapter that
  produces the same `list[list[int]]` shape as MMH's `matches`.

Exit criteria: the v4 data extensions reduce `failed_extraction` to ≤
10 on a fresh 500-char batch. Never violates the geometry skill — the
placement side is unchanged.

## v5 — Anchor-binding second authoring path

Scope (from the original scene-graph spec, D6): implement anchor-binding
as an alternative to measured source_stroke_indices. Some chars compose
better from anchor relations ("inner.center → outer.center"; "A.right →
B.left + gap") than from a raw stroke partition. The compose algorithm
already has the interface (`anchor_bindings` field on the plan) — what's
needed is the solver that turns anchor relations into an affine.

Exit criteria: a new `test_anchor_binding_compose.py` covers at least 3
worked examples (simple align_x / center_on / attach). No existing
tests regress.

## v6 — Learned layout adapter (deferred; IF-Font / CCFont-style)

Scope: a fourth authoring adapter that predicts `InstancePlacement`
trees from (component list, style ref) without MMH matches, by training
on existing measured records. Structural reference: `vault/references/
papers/2026-04-22-ccfont.md` and `2026-04-22-vector-component-composition.md`.
Adopts only the measured-transform output shape; not the preset-style
categorical inputs.

Exit criteria: learned adapter reproduces v1's 4 seed char transforms
within ≤ 0.05 mean affine-element distance against measured ground
truth.

## vN — Styling stage (separate project)

The scene-graph spec §pipeline-split defines this as a separate stage.
Stays deferred until v1–v4 are solid.

## Failure taxonomy ↔ version mapping

Current at-scale report (196 filled / 4808, after v1 + the v1-finish
cleanup in commit `e6b82f9`):

| Class | Count | Sample | Assigned to |
|---|---|---|---|
| A: cjk > MMH partition arity (nested) | 29 | 丐 乙 兆 刁 剎 力 | v3 |
| B: MMH matches absent for host | 0 | — | (would be v4 if surfaces) |
| C: host not in MMH graphics | 7 | 縲 臏 蓀 迆 鉑 鍰 | v4 |
| D: component not in MMH graphics | 28 | 乏 侖 前 叉 商 天 | v4 |
| E: needs_review (iou below gate) | 5 | 匠 威 式 街 褻 | v2 |

Nothing in this taxonomy requires reintroducing preset vocabulary, a
preset-style categorical flag, or a compose-time weight calculation. The
geometry skill stays binding.

## Not-in-roadmap (explicit non-goals)

- Preset adapters as data model (forbidden — see skill).
- Fallback to global slot weights under any circumstance.
- Categorical flags (`is_radical: bool`) to patch the measurement path.
- Compose-time weight derivation as a substitute for storing measured
  transforms on the plan.

## Operational note

Every new failure mode should land in this document under the version
that will solve it, even if that version isn't scheduled yet. If no
version would solve a failure, that itself is a signal: either the spec
needs to grow a new version or the failure is a one-off that should be
closed as "won't fix" with a recorded reason.
