# Plan 13 — v2 full 4808 coverage + batch styling

Finishes v2: every char in the Taiwan 4808 list either reaches `verified`
or has a recorded per-char reason in `vault/references/v2-unresolved.md`.
Every verified char is stylable through the Plan 12 ComfyUI bridge.

Workflow file: `.archon/workflows/plan-13-v2-full-4808-coverage.yaml`.

## Tasks

### Task A — Recursive decomposition

1. Extend `bulk/planner.plan_char`: when `top_level_partition(matches)`
   has fewer buckets than `len(components)`, descend into
   `nested_partition(matches, idx=k)` for each top-level bucket and
   re-check arity at that level. If still short, keep descending.
   Emit `GlyphNodePlan` subtrees with `mode="refine"` for the inner
   levels.
2. New test `test_planner_recursive_partition.py`: 森 (already works)
   plus 3 known class-A chars (丐 乙 兆) successfully plan with nested
   partitions.
3. Commit: `feat(py): planner descends into MMH nested partitions`.

### Task B — animCJK graphics + dictionary loader

1. New module `project/py/src/olik_font/sources/animcjk.py`: parallel
   to `sources/makemeahanzi.py`. Parses `ref-projects/animCJK/
   graphicsZhHant.txt` + `dictionaryZhHant.txt` into the same `MmhChar`
   / `MmhDictEntry` shape. Installed as a data snapshot under
   `project/py/data/animcjk/` (copy-on-build from ref-projects so the
   lineage is recorded).
2. New `sources/unified.py` merges MMH + animCJK: a unified
   `char_graphics_lookup(char) -> MmhChar | None` that tries MMH
   first, then animCJK. Log the source of every miss during a bulk
   run.
3. `bulk/batch.run_batch` uses the unified loader.
4. Test `test_animcjk_source.py`: loads the snapshot, covers at least
   5 class-C chars (縲 臏 蓀 迆 鉑 鍰) successfully.
5. Commit: `feat(sources): animCJK graphics + dictionary loader`.

### Task C — Component carving

1. New module `project/py/src/olik_font/prototypes/carve.py`: when the
   unified lookup has no standalone entry for a component name, find
   a containing parent via cjk-decomp (any host where this component
   appears), pull the parent's MMH graphics, partition via its
   `matches`, and carve the component's strokes. Cache the carved
   result in `project/py/data/carved_components.json` so subsequent
   runs are fast.
2. `bulk/planner._extract_canonical_prototype` falls back to
   `carve_component` on RuntimeError.
3. Test `test_component_carving.py`: the 28 class-D chars' components
   resolve via carving (sample 乏 侖 前 叉 商 天).
4. Commit: `feat(prototypes): carve components from containing hosts`.

### Task D — Per-prototype variant cap

1. Replace `--max-variants-per-proto N` with a config file at
   `project/py/data/variant_caps.yaml`: per-prototype or per-name-glob
   caps, default 10. Example: `木: 20`, `口: 20`, `default: 10`.
2. Thread the per-prototype cap into `reuse.decide_prototype`.
3. Test `test_per_proto_variant_cap.py`.
4. Commit: `feat(py): per-prototype variant cap config`.

### Task E — cjk-decomp overrides

1. New file `project/py/data/cjk_decomp_overrides.yaml` keyed on the
   character, value = `{operator, components}` replacing the upstream
   cjk-decomp entry. Empty initially.
2. `bulk/batch._load_cjk_entries` applies overrides on top of the
   upstream file; logs every override at load time.
3. Workflow rule: when a char keeps failing for partition-shape
   reasons despite correct MMH data, add an override rather than
   patching the planner to tolerate the mismatch.
4. Commit: `feat(data): cjk-decomp overrides file`.

### Task F — Scale styling to 4808

1. Extend `olik style` with a `--all-verified` flag: pulls every
   `verified` char from the DB, runs the Plan 12 ComfyUI bridge in
   batch.
2. Concurrency cap: one ComfyUI job in flight at a time (to match
   single-worker ComfyUI install). Expose `--max-concurrent N` for
   future.
3. Persist a `styled-output/manifest.json` with `(char, style, seed,
   output_path, generated_at)` so reruns can skip.
4. Commit: `feat(cli): olik style --all-verified`.

### Task G — Full-coverage auto run + visual verdict sweep

1. Bulk extract: `olik extract auto --count 4808 --seed 2026
   --max-variants-per-proto 10` (or loop across multiple seeds to
   fill the pool).
2. After extraction, `olik extract report` should show verified ≥
   4700.
3. For remaining non-verified chars, run
   `scripts/v2_unresolved_report.py` which writes
   `vault/references/v2-unresolved.md` with per-char reason + sample
   MMH geometry + sample cjk-decomp entry.
4. Styling sweep: `olik style --all-verified --styles ink-brush
   --seeds 1 --out styled-output/v2-sweep/` — first-pass, one style,
   one seed, just to exercise the batch at scale.
5. Kimi verdict sweep (sample of 200 chars):
   `scripts/verdict_v2.py` invokes kimi on a random 200-char sample
   checking BOTH geometry (vs MMH) AND styling legibility. Aggregate
   pass rate ≥ 95%.
6. Apply tag `v2-full-coverage`.
7. Commit each of these artifacts incrementally (one commit per item
   at minimum).

## Acceptance

- `olik extract report`: verified ≥ 4700 of 4808.
- `vault/references/v2-unresolved.md` lists every non-verified char
  with reason.
- Every verified char has at least one styled PNG in
  `styled-output/v2-sweep/`.
- Kimi 200-sample sweep ≥ 95% pass rate.
- Zero preset vocabulary residuals (same gate as Plan 11).
- Tag `v2-full-coverage` applied and pushed.
